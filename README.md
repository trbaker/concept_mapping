

Concept Mapper is a browser-based studio for building concept maps — the diagram style where ideas are drawn as boxes or ovals and joined by labeled arrows, so that each "concept → linking word → concept" reads as a short sentence, or proposition. It runs entirely as a single self-contained HTML file with no install, login, or server: everything happens locally in the browser. The workspace is a dotted-paper canvas with five tools — select, box, oval, link, and text — plus connectors that attach to nodes and re-route themselves automatically when you move things around. You can rename anything by double-clicking, recolor and resize from the side panel, undo and redo your steps, and finish by exporting the finished map as a JPG image or saving an editable file you can reopen later.

The interface follows a warm-neutral design system: a light, monochrome palette of soft taupe-grays with a single clay accent reserved for the active tool, paired with the Bricolage Grotesque and Hanken Grotesk typefaces and generous rounded corners. Node colors are offered as a tonal ramp — from a clean white card through deeper inks — rather than bright hues, keeping maps calm and readable while still letting you distinguish clusters. A built-in guide explains the idea of propositions and cross-links for students new to the technique.

Because the app is meant to be published openly, it takes the file-opening feature seriously on the security side. Uploaded maps are never trusted as-is: a strict validator rebuilds every shape field by field, whitelists the allowed types, clamps all numbers and sizes, validates colors, and caps both file size (30 KB) and shape count (200). The properties panel is built with safe DOM methods rather than raw HTML so imported data can't inject markup, and the page ships with a Content-Security-Policy that allows only its own verified script — closing off the usual cross-site-scripting paths a hostile file might try to exploit.
